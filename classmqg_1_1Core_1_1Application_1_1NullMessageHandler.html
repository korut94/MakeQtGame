Detailed Description

The Application::NullMessageHandler class defines a null service for messaging task.

In way to avoid temporary states where the message handler service in the Application class is no setted (that should involve in a dereferentation of a null pointer), the NullMessageHandler provides a null service where in any method nothing is done.

Note
This class not should be use in the application directly, since the Application class worries about the instantiation of a null service when requirement. For example:
Application::provide(nullptr); // Application will use a NulMessageHandler
